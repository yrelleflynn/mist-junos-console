# **The Self-Driving Network Hackathon**

### **Build the Automation That Runs the Network**

**Goal:**

Create a **production-ready automation, script, or application** that moves Mist-managed networks closer to a **self-driving operating model**.

The winning projects will **detect problems, make decisions, and take action automatically** - reducing the need for manual network operations.

Think of this as:

**"What would the network fix by itself if it could?"**

# **Core Challenge**

Build a tool that enables **self-driving operations** by doing one or more of the following:

• **Automatically detect issues**

• **Diagnose root cause**

• **Recommend corrective actions**

• **Automatically remediate problems**

• **Continuously optimize network performance**

Solutions must work with **Mist APIs, telemetry, or automation capabilities**.

# **What "Self-Driving" Means in This Hackathon**

Submissions should implement at least **one level of autonomous operation**:

### **Level 1 - Intelligent Detection**

Automatically identify problems using Mist data.

Examples:

- Detect abnormal client behavior
- Identify misconfigured sites
- Spot poor roaming environments
- Find APs causing RF problems

### **Level 2 - Automated Diagnosis**

Analyze Mist telemetry to determine root cause.

Examples:

- Determine whether issues are RF, client, switch, or WAN related
- Correlate events across sites
- Identify patterns across multiple organizations

### **Level 3 - Autonomous Action**

Automatically fix or mitigate problems.

Examples:

- Adjust configurations automatically
- Trigger remediation workflows
- Restart problematic devices
- Trigger alerts with automated context

# **Required Characteristics**

All submissions must be:

### **Production Ready**

The tool must be usable by customers immediately.

Required:

- Error handling
- Documentation
- Setup instructions
- Example outputs
- Must be Secure

### **Broadly Applicable**

The tool should work across **many customer environments**, including:

- Retail
- Campus
- Healthcare
- Education
- Branch networks

Solutions designed for **hundreds or thousands of sites** score higher.

### **Mist-Native**

Projects must leverage **Mist platform capabilities**, such as:

- Marvis
- Mist APIs
- Telemetry data
- Event streams
- Network analytics

# **Deliverables**

Each team must submit:

### **1\. Working Code**

Hosted in a Git repository.

### **2\. Documentation**

Include:

- Problem statement
- Architecture
- Setup instructions
- Example output

### **3\. Presentation and Demo (10-15 minutes)**

Show:

- The operational problem
- The solution
- Real output or actions taken

### **4\. Customer Impact Summary**

Explain:

**Why customers would deploy this immediately.**

# **The Ultimate Test**

The winning tool should make customers say:

**"This is exactly what a self-driving network should do."**

### **Questions & Guardrails**

To ensure fairness and avoid overwhelming the Product Management team, all hackathon questions will follow a **centralized process**.

**Submission Process**

- All questions must be submitted through the **official Hackathon Question Form**.
- Questions may be submitted **at any time during the hackathon**.
- Questions will be answered **in the order they are received**.

**Shared Answers**

- All questions and responses will be published in a **shared public spreadsheet**.
- This ensures **all teams receive the same information and guidance**.
- Teams should **review the spreadsheet before submitting new questions** to avoid duplicates.

**Communication Guardrails**

- **Do not reach out to PLMs individually** with hackathon questions.
- PLMs (e.g., Thomas Munzer) are not permitted to actively participate in or engage with any team activities.
- All technical or product-related questions must go through the **question form**.
- This process ensures **consistent answers, equal access to information, and minimal disruption to ongoing work**.

Participants may **self-organize and form teams** for the hackathon.

**Team guidelines:**

- Teams may consist of **1 to 3 members**
- Participants are free to **self-select teammates**
- **Maximum team size is 3 people**
- All participants must be members of the **Mist Field PLM team**
